Objectif. Plateforme unique couvrant le flux bout-en-bout d'un laboratoire BTP : demande → prélèvement → essai → résultats → rapport → facturation, avec volet géotechnique (missions, forages, lithologie) et saisie terrain mobile.

État actuel. MVP fonctionnel en production (Laravel 11 API + React 18 SPA + mobile Expo). Couvre CRM, chantiers/sites, devis, commandes, échantillons, résultats, rapports PDF, facturation FR/MA (TVA 20 %, MAD, ICE/RC/Patente).

Cap produit. Industrialiser la traçabilité (ISO 17025 / SNIMA), outiller les équipements (étalonnages), ouvrir un portail client et connecter les instruments (V2).

6. Planning & feuille de route

6.1 Phasage global

MVP · 6–10 semaines

Lot 1 — Référentiel & essais livré

  • Clients, agences, sites, missions, forages
  • Catalogue essais + normes
  • Commandes → échantillons → résultats
  • Rapports PDF + signature
  • Devis / factures FR + MA
V1 · 10–16 semaines

Lot 2 — Industrialisation en cours

  • Planning techniciens + équipements
  • Gestion étalonnages (certificats, échéances)
  • Non-conformités + actions correctives
  • Audit trail renforcé (diff valeurs)
  • Bilingue FR/AR (interface + PDF)
V2 · 16+ semaines

Lot 3 — Ouverture & BI planifié

  • Connecteurs instruments (presse, balance, CSV/RS232)
  • Portail client PDF + paiement en ligne
  • BI / KPI (Metabase, Superset)
  • Multi-labos / multi-sociétés
  • IA : détection anomalies résultats

7.4 Dette technique à adresser

  • Supprimer ou archiver le dossier services/ (anciens microservices Node non utilisés).
  • Harmoniser le JSON meta sur toutes les entités (schema JSON par entité + validation).
  • Mutualiser les templates PDF (3 templates redondants actuellement, layout_config à étendre).
  • Migration SQLite → PostgreSQL en prod (index JSON, performance, concurrency).
  • Normaliser les statuts (enum PHP partagé entre Laravel et React au lieu de strings libres).
